# "Unresolved reference 'layout'" 오류 해결 가이드

## 🔍 문제 진단

이 오류는 `R.layout.activity_main`을 찾을 수 없을 때 발생합니다. 다음을 확인하세요:

### 1단계: 파일 위치 확인

**Windows 탐색기에서 확인:**
1. `C:\Users\BSA\AndroidStudioProjects\RGBSensorBLE\app\src\main\res\layout\` 폴더로 이동
2. `activity_main.xml` 파일이 있는지 확인

**없다면:**
- `layout` 폴더가 없으면 생성
- `activity_main.xml` 파일이 없으면 `yellow` 폴더에서 복사

### 2단계: Android Studio에서 확인

1. **Project 뷰 확인**
   - 왼쪽 상단의 **Project** 탭 클릭
   - `app > src > main > res > layout` 경로 확인
   - `activity_main.xml` 파일이 보이는지 확인

2. **Android 뷰 확인**
   - 왼쪽 상단의 **Android** 탭 클릭
   - `app > res > layout` 경로 확인

### 3단계: 즉시 해결 방법

#### 방법 A: Gradle 동기화 (가장 먼저 시도)

1. **File > Sync Project with Gradle Files**
2. 동기화 완료 대기
3. 오류가 사라졌는지 확인

#### 방법 B: 프로젝트 정리

1. **Build > Clean Project**
2. 완료 후 **Build > Rebuild Project**
3. 오류 확인

#### 방법 C: 캐시 무효화

1. **File > Invalidate Caches / Restart**
2. **Invalidate and Restart** 클릭
3. Android Studio 재시작 후 Gradle 동기화

#### 방법 D: 파일 수동 복사 (파일이 없는 경우)

1. Windows 탐색기 열기
2. `C:\Users\BSA\AndroidStudioProjects\yellow\activity_main.xml` 복사
3. `C:\Users\BSA\AndroidStudioProjects\RGBSensorBLE\app\src\main\res\layout\` 에 붙여넣기
4. Android Studio에서 **File > Sync Project with Gradle Files**

### 4단계: build.gradle 확인

`app/build.gradle` 파일을 열어서 다음을 확인:

```gradle
android {
    namespace 'com.example.rgbsensorble'  // 이 줄이 있어야 함
    compileSdk 34
    
    // ... 나머지 설정
}
```

**중요:** `namespace`가 있어야 R 클래스가 올바르게 생성됩니다.

### 5단계: 패키지 확인

`MainActivity.kt` 파일의 첫 줄 확인:
```kotlin
package com.example.rgbsensorble
```

이 패키지가 `build.gradle`의 `namespace`와 일치해야 합니다.

## ✅ 체크리스트

다음을 모두 확인하세요:

- [ ] `app/src/main/res/layout/activity_main.xml` 파일 존재
- [ ] `app/build.gradle`에 `namespace 'com.example.rgbsensorble'` 있음
- [ ] `MainActivity.kt`의 패키지가 `com.example.rgbsensorble`
- [ ] Gradle 동기화 완료
- [ ] 프로젝트 정리 및 재빌드 완료

## 🚨 여전히 해결되지 않는 경우

1. **프로젝트를 완전히 닫고 다시 열기**
   - File > Close Project
   - File > Open > RGBSensorBLE 폴더 선택

2. **새 프로젝트로 다시 시작**
   - 방법 1의 스크립트 사용 (`프로젝트_구조_생성.ps1`)
   - 현재 `yellow` 폴더에서 직접 프로젝트 생성

3. **Android Studio 재설치** (최후의 수단)

## 💡 예방 방법

앞으로 이런 문제를 방지하려면:
- 파일 복사 후 반드시 Gradle 동기화
- 프로젝트 구조를 정확히 확인
- 자동화 스크립트 사용 권장
